# Round-efficient Fully-scalable MPC algorithms for k-Means 

Shaofeng H.-C. Jiang ∗ Yaonan Jin † Jianing Lou ‡ Weicheng Wang §

Abstract 

We study Euclidean k-Means under the Massively Parallel Computation (MPC) model, focusing on the fully-scalable setting. Our main result is a fully-scalable O((log n/ log log n)2)-approximation in O(1) rounds. Previously, fully-scalable algorithms for k-Means either run in super-constant O(log log n · log log log n) rounds, albeit with a better O(1)-approximation [Cohen-Addad et al., SODA’26], or suffer from bicriteria guarantees [Bhaskara and Wijewar-dena, ICML’18; Czumaj et al., ICALP’24]. Our algorithm also gives an O(log n/ log log n)-approximation for k-Median , which improves a recent O(log n)-approximation [Goranci et al., SODA’26], and this o(log n) ratio breaks the fundamental barrier of tree embedding methods used therein. Our main technical contribution is a new variant of the MP algorithm [Mettu and Plaxton, SICOMP’03] that works for general metrics, whose new guarantee is the Lagrangian Multiplier Preserving (LMP) property, which, importantly, holds even under arbitrary distance distortions. Allowing distance distortion is crucial for efficient MPC implementations and useful for efficient algorithm design in general, whereas preserving the LMP property under distance distortion is known to be a significant technical challenge. As a byproduct of our techniques, we also obtain an O(1)-approximation to the optimal 

value in O(1) rounds, which conceptually suggests that achieving a true O(1)-approximation (for the solution) in O(1) rounds may be a sensible goal for future study.

The classical (Euclidean) k-Means clustering problem was proposed in the 1950s [Llo82], and has since become one of the most fundamental problems in data analysis and machine learning. The study of k-Means , and more generally ( k, z )-Clustering , has been a central combinatorial optimization problem in the design and analysis of algorithms; formally, for a fixed z ≥ 1, given as input an integer k ≥ 1 and a dataset P ⊆ Rd, ( k, z )-Clustering asks for a set of k centers

The ( k, z )-Clustering problem captures both k-Means (when z = 2) and another widely studied 

k-Median problem (when z = 1). We address the computational challenges of (Euclidean) ( k, z )-Clustering on large-scale and high-dimensional datasets, focusing on the Massively Parallel Computation (MPC) model. The MPC model was introduced in [KSV10] and is known to serve as an abstraction of several practical parallel computing frameworks, including MapReduce [DG08], Hadoop [Whi15], Spark [ZCF +10], and Dryad [IBY +07]. In the general MPC model, N input elements are distributed arbitrarily across a set of machines, each with sublinear local memory s = o(N ) to store elements and bits. The computation proceeds in synchronous rounds . In each round, every machine may exchange messages with any other machine, subject to the constraint that the total number of elements and bits sent/received is at most s, and can perform arbitrary local computation. The output of the algorithm may be distributed across the machines. We aim to devise MPC algorithms that achieve the following ideal efficiency goals: a) fully-scalable , i.e., they work with any (small) local memory s = N ε for any constant ε ∈ (0 , 1); and b) round-efficient , specifically aiming for constant round complexity. Both of these ideal goals have been widely pursued in the study of MPC algorithms [ANOY14, ASS +18, ASZ19, GU19, CDP21, EMMZ22, AAH +23, CC23, CGJ +24, CZ24, JMNZ24, ABJ +25, CGGJ25, CKP26, GJK +26], and achieving them simultaneously has often proven to be significantly difficult. In our specific context of Euclidean clustering, we model each coordinate of a point in Rd as a storage element. Thus, the input is of size O(nd ), and a fully-scalable algorithm must work with local memory s = ( nd )ε for any 

ε ∈ (0 , 1). This particularly allows s = o(k), which means we cannot afford to store a full solution on any single machine, and also renders popular sketch-based techniques, such as coresets [HM04], less useful, as these methods inherently require local memory Ω( k) [HV20, CLSS22, HLW24]. Recent years have witnessed a surge of interest in designing fully-scalable algorithms for cluster-ing problems [BW18, BEFM21, CLN +21, CMZ22, CCM23, CGJ +24, CGGJ25, CKP26, GJK +26]. Despite such progress, a significant research gap remains for k-Means : no fully-scalable algorithm that approximates k-Means is known to achieve ideal round efficiency (i.e., O(1) rounds). Indeed, existing O(1)-round (fully-scalable) algorithms either guarantee only a bi-criteria approximation (whose solution may use more than k centers to attain the claimed ratio) [BW18, CGJ +24], or work only for special types of input [CMZ22]; recently, [CKP26] obtained a constant-approximation that works for general inputs, but this comes at the cost of a super-constant round complexity of 

O(log log n · log log log n). Technically, this research gap is attributed to fundamental barriers in existing frameworks. In particular, the algorithm of [CKP26] is based on the Jain-Vazirani framework [JV01] where one of its crucial steps involves computing graph ruling sets . Unfortunately, graph problems (not just ruling sets) often possess intrinsic complexities that make them difficult to solve within constant rounds, 1as evidenced by several (conditional) hardness results [GKU19, CDP24]. This is the primary bottle-neck behind the super-constant round complexity in [CKP26], and suggests that a more thorough exploitation of the Euclidean structure may be necessary to achieve constant rounds. On the other hand, a series of works [CLN +21, AAH +23, GJK +26] leverage tree embeddings , which are a powerful tool in Euclidean spaces with applications beyond clustering, to achieve a polylog( n)-approximation for k-Median . However, tree embeddings are well known to be ill-suited for handling higher powers of distances, and thus struggle to extend beyond k-Median to general ( k, z )-Clustering ; even if tree embeddings could handle ( k, z )-Clustering , they still suffer from an inherent Ω(log n) lower bound on distance distortion [AKPW95, Bar98], which prevents this approach from surpassing an 

O(log n) approximation ratio, even for k-Median . In sum, to bridge the gap in designing fully-scalable constant-round algorithms for Euclidean ( k, z )-Clustering , it seems essential to seek a new framework that is more general and better exploits the Euclidean structure. 

1.1 Our Results 

To address this gap, we propose a new framework for clustering that works for general metrics and can be implemented efficiently in Euclidean spaces for MPC, using known geometric primitives such as approximate range queries and metric ruling sets. This leads to our main theorem (The-orem 1.1): the first round-efficient fully-scalable algorithm for Euclidean ( k, z )-Clustering that achieves a nontrivial approximation ratio, 1 and the ratio for k-Median (z = 1) breaks the tree em-bedding barrier. Our techniques also lead to an O(1)-approximation for estimating the value of the optimal solution (Theorem 1.2). Although this only estimates the value, this O(1)-approximation is nevertheless the first of its kind in the literature, and conceptually it may reveal that a true 

O(1)-approximation (for the solution) is a sensible goal for future study. We state our main result in Theorem 1.1. For ( k, z )-Clustering , value estimation is already nontrivial. This stands in contrast to other fundamental problems such as Facility Location and Minimum Spanning Tree , which admit efficient MPC value estimation by adapting (linear-sketch-based) streaming algorithms [CJK +22, CCJ +23] (and this reduction was also observed in e.g., [CGJ +24, JMNZ24]). However, for ( k, z )-Clustering , this approach is unlikely to yield fully-scalable algorithms, since any streaming algo-rithm with finite approximation ratio requires Ω( k) space, even to differentiate whether the optimal is 0 or not. Conceptually, our Theorems 1.1 and 1.2 reveal a potential gap between value estimation and finding approximate solutions. Such a “value v.s. solution” gap is also seen in various settings for several related fundamental problems; notable examples include Facility Location in the offline setting [GIV01, BCIS05], as well as Minimum Spanning Tree in both the offline setting [CS09, HIM12] and the MPC setting [CCJ +23, JMNZ24]. Therefore, bridging this gap for clustering in MPC is an interesting open question. 

1.2 Technical Overview 

Our MPC algorithm is based on a new offline framework for clustering that works for general metrics , and can be implemented in MPC via three subroutines (apart from basic operations such as sorting and broadcasting): (a) a metric ruling set algorithm, (b) an approximate range query algorithm, and (c) an approximate nearest neighbor algorithm. Our main results are then obtained by plugging in existing round-efficient algorithms for these subroutines in Euclidean spaces [CGJ +24, CGGJ25]. Moreover, any further improvements in these subroutines, in particular for metric ruling sets, would directly lead to improved bounds for our algorithms. At a high level, this framework consists of two steps: it first computes a fractional clustering solution, and then applies a rounding procedure to convert it into an integral solution (i.e., a set of centers). 

Fractional Algorithm. The algorithm that constructs a fractional solution is our main tech-nical contribution, and it immediately implies Theorem 1.2 for O(1)-approximate value estima-tion. The key ingredient of this fractional algorithm is a new variant of the Mettu-Plaxton 3(MP) algorithm [MP03], which has been widely adapted to various sublinear settings for cluster-ing [ARS03, BCIS05, GLS14, CJK +22, BCG +24, BGJ +24, CGJ +24]. As usual, our variant of MP aims to solve the so-called (power-z) facility location problem – a (Lagrangian) relaxation of ( k, z )-Clustering , but importantly, it additionally satisfies the Lagrangian Multiplier Preserving (LMP) 

property for all z ≥ 1, even on arbitrarily distorted distances which may not form a metric (not even symmetric); see Algorithm 1 and Lemma 3.1. Indeed, LMP is a widely considered property that turns approximations to power-z facility location to ( k, z )-Clustering via the so-called bi-point rounding procedure [JV01, JMS02, JMM +03, LS16, BPR +17, CGLS23, GPST23, BCG +24]. Moreover, being robust to distance distortion is a natural yet useful requirement, as it is a common strategy to introduce distance distortions to trade for efficiency. More specific to our MPC Eu-clidean setting, this allows us to achieve a round-efficient MPC implementation via an approximate range query [CGJ +24] subroutine (which distorts the distance). However, while certain variants of MP are known to possess LMP [ARS03, BCG +24], they may not hold even with tiny distance distortions. In fact, handling distorted distances while preserving LMP is a general challenge that entails significant technical difficulties, as we discuss below. 

Challenges of Preserving LMP under Distance Distortion. We begin with some back-ground. Let us focus the discussion on the (integral) power-z facility location problem, since this already captures the main challenges of the fractional case. In power-z facility location, instead of imposing a strict bound k on the number of centers, we are given a parameter λ > 0 as the opening cost per center, and the goal is to find a set F ⊆ P minimizing λ|F | + cost( F ), where λ|F | is called the opening cost and cost( F ) (identical to the ( k, z )-Clustering cost) is called the connection cost. We say that a solution F is an LMP α-approximation for some α ≥ 1 if cost( F )+ αλ |F | ≤ α·OPT fl ,where OPT fl denotes the optimal value of the facility location problem. This is clearly a stronger guarantee than a standard α-approximation. While for non-LMP approximations to (power-z) facility location, an α-factor distortion in distances often translates to an O(αz ) factor in the final approximation ratio, the stronger LMP property is more fragile: even a 1 + ε error can cause the resulting solution to be no longer LMP. Specifically, consider an LMP α-approximate solution F computed using the (1 + ε)-distorted distances, which means 

gcost( F ) + αλ |F | ≤ α ]OPT fl , (1) where both gcost and ]OPT fl are evaluated on the (1 + ε)-distorted distances. However, it is possible that this solution F is not LMP with respect to the original distances. Indeed, while the opening cost 

λ|F | does not change with respect to distance distortion, ]OPT fl and gcost( F ) may change by a factor of, say, 1 + ε, and therefore the guarantee (1) only implies (1 + ε) cost( F ) + αλ |F | ≤ (1 + ε)α OPT fl .This still leaves a gap from the LMP guarantee we actually want, i.e., cost( F ) + α′λ|F | ≤ α′ OPT fl ,for (finite) α′ ≥ 1; in fact, this gap cannot be bridged for a generic solution F , as shown by the following counterexample. 

Example. Consider z = 1 and λ = 1 + ε (for any given ε > 0). Let the dataset consist of only two points x, y with dist( x, y ) = 1, and distorted distance gdist( x, y ) = 1 + ε. On the original distances, the optimal value is OPT fl = 2 + ε. On the distorted distances, one optimal solution (which is clearly LMP 1-approximate) is to open both points as facilities, i.e., F = {x, y }. However, on the original distances, since cost( F ) = 0, for any finite α′ ≥ 1, we have cost( F ) + α′λ|F | = α′ · 2(1 + ε) > α ′ · (2 + ε) = α′ · OPT fl .

Therefore, F is not an LMP solution with any finite approximation ratio on the original distances. 4Since LMP does not hold under distance distortions in general, one must utilize the properties of a specific algorithm, or a specialized way of distorting distance. Indeed, a recent work [CKP26] manages to implement the seminal Jain-Vazirani algorithm (JV) [JV01] in MPC, but it only works through careful distance approximations (e.g., using Euclidean spanners) and highly sophisticated modifications to JV. Instead, our focus is on the MP algorithm, which is very different from JV, and our goal is stronger in that any distance distortion works (which no longer requires careful design of distance approximation). 

Fractional MP with LMP, Robust to Distance Distortions. The first variant of the MP algorithm that has the LMP property was introduced in [ARS03], and a recent work [BCG +24] adapts the MP algorithm to the fractional setting and also establishes the LMP property. However, the LMP property established in these works is not known to be robust to distance distortions, and another limitation is that they only apply to the case z = 1. While our algorithm for z = 1 is based essentially on the same algorithm as in [BCG +24], our novelty lies in the analysis: we show that the LMP property of this algorithm is robust to distance distortions, and we further generalize this algorithm and the analysis to general z ≥ 2 (where we also encounter a fundamental challenge). Rounding Algorithm. For the rounding part, the main technical idea is to implement an offline rounding process, which is a modified version of [CGTS99], via metric ruling sets (Definition 5.3). 5Ruling sets (or the closely related maximal independent set) are often used as a “pruning” step in (variants of) primal-dual/JV-based algorithms for facility location [JV01, ARS03, ANSW20, CEMN22, GOR +22, CKP26]. However, in these works, the ruling set is typically defined with respect to hop distances in certain (unweighted) graphs, which is conceptually different from our notion of a metric ruling set, as ours is defined with respect to distances in the underlying metric (in our case, Euclidean space). This notion of metric ruling set was introduced in a recent work [CGGJ25], where it is shown that, in Euclidean space, it admits a fully-scalable O(1)-round MPC algorithm by exploiting the Euclidean structure via geometric hashing techniques. On the other hand, graphs generally lack such Euclidean structure, and one often has to rely on ruling set algorithms for general graphs, which typically require super-constant rounds [GU19, KPP20]. This is also a key reason why a recent work [CKP26] requires ω(1) rounds; while it may be possible to replace the graph ruling sets in [CKP26] by metric ruling sets, it would be unlikely to yield (significantly) better results than ours, since both our approximation ratio and round complexity match the state-of-the-art metric ruling set bounds (up to constant factors). We provide a more detailed overview of the rounding algorithm in Section 1.2.2.

Generalizing to z ≥ 2: New Steps. Finally, we discuss how to generalize the algorithm and analysis to z ≥ 2. The first step is to generalize the definition of rp’s, and we make use of a natural generalization from [GLS14, CGJ +24]. While the above argument for er generally works, it actually relies on one key premise: exact r(β) is LMP (i.e., whether (3) holds), which has been proved in [BCG +24] for z = 1. Unfortunately, this premise is not known to hold for z ≥ 2. Therefore, we need an added step to prove that r(β) is LMP. To this end, we try to generalize the analysis of [BCG +24] from the z = 1 case to z ≥ 2. The key step in their approach is to define an assignment x′ that is feasible with respect to r(β), and to show that its connection cost, which is no larger than cost( r(β)), is good enough for LMP. 1.2.2 Proof Overview: MPC Rounding Algorithm 

We now discuss our MPC rounding algorithm, which, given a dataset P and a fractional solution 

y ∈ RP 

> ≥0

with ∥y∥1 = k (opening each p with amount yp as a center), returns an integral set C ⊆ P

with |C| ≤ k and comparable clustering cost. We focus on the z = 1 case. At a high level, our MPC rounding algorithm follows the framework of [CGTS99]. This frame-work is designed for the offline setting, and many of its key steps are inefficient for MPC imple-mentation. We re-cast these inefficient steps as a non-uniform variant of the ruling set problem: the distance requirements vary across points. We further reduce this non-uniform variant to the standard metric ruling sets, which allows us to leverage recent progress [CGGJ25] to obtain an 

O(1)-round rounding algorithm. The framework of [CGTS99] consists of three main steps: (i) sparsification : sparsify the dataset 

P into a (weighted) subset of size at most 2 k, where each point is treated as a candidate center; (ii) 7partial rounding : open each candidate center with amount 1 /2 (half-open) or 1 (fully open), while ensuring a total opening of at most k; and (iii) final rounding : select at most half of the half-open centers to be fully open, and thus there are at most k fully open centers. The final integral solution is then the set of these fully open centers. Below, we explain each step in more detail. 

Step 1: Sparsification. The algorithm in [CGTS99] sparsifies P as follows: initially, every point is unmarked; they process the points p ∈ P in increasing order of their contribution to cost( y), denoted by cost( p, y), and mark p if every point within distance O(1) · cost( p, y) is unmarked. The set of marked points (with appropriate weights) then forms the sparsified dataset. We reinterpret their goal of sparsification as constructing the following variant of a ruling set: find a subset Q such that (a) for every distinct p, q ∈ Q, dist( p, q ) ≥ Ω(1) ·max {cost( p, y), cost( q, y)},and (b) for every p ∈ P , dist( p, Q ) ≤ O(1) · cost( p, y). Our solution is based on a key observation: when all points have the same contribution cost( p, y) to the fractional solution y, the problem becomes equivalent to a metric ruling set problem. Motivated by this, we group points with similar values of cost( p, y) and construct a metric ruling set for each group in parallel. With careful handling across groups, we can obtain the desired set Q (see Section 5.1.1 for details). 

Step 2: Partially Rounding. The algorithm of [CGTS99] for the second step involves only basic operations, such as approximate nearest neighbor queries and sorting, which can already be efficiently implemented in MPC using [CGJ +24] and [GSZ11], respectively. 

Step 3: Final Rounding. After the first two steps, we obtain a partially rounded solution, where each open center is either half-open or fully open; in the final step, we aim to select at most half of the half-open centers to be fully open. For this step, the algorithm of [CGTS99] relies on vertex coloring of forests and is thus inefficient in the MPC setting. We reinterpret their algorithm as solving the following variant of the dominating set problem: let H denote the half-open centers; select H′ ⊆ H such that (i) for every p ∈ H, dist( p, H ′) ≤ O(1) · dist( p, H \ { p}); and (ii) 

|H′| ≤ | H|/2. We make the following observation. When every point has the same distance τ > 0to its nearest neighbor, it suffices to select a 2 τ -ruling set of H as H′ (which is also a dominating set); moreover, the separation property of ruling sets, i.e., ∀p̸ = q ∈ H′, dist( p, q ) > 2τ , ensures that 

|H′| ≤ | H|/2. Building on this observation, we follow a similar approach as in the first step: we group the points in H by nearly equal nearest-neighbor distances and construct a ruling set for each group in parallel, with some differences in the implementation details (e.g., ensuring |H′| ≤ | H|/2requires different handling across groups). 

1.3 Related Work 

Fully-scalable MPC algorithms have also been studied for fundamental problems in Euclidean spaces other than k-Means . In the context of clustering, closely related problems include k-Center [BEFM21, CCM23, CGGJ25] and Facility Location [CGJ +24]. To date, the best-known fully-scalable algorithm for k-Center achieves an O(log n/ log log n)-approximation in a constant number of rounds [CGGJ25], and in low dimensions, this approximation ratio can be further improved to 2+ ε (or 1+ ε for a bi-criteria solution). For Facility Location , the streaming algorithm of [CJK +22] can be directly translated into an O(1)-round MPC algorithm with an O(1)-approximation, but this only applies to estimating the optimal value. More recently, [CGJ +24] resolved the solution version of Facility Location by achieving the same O(1)-approximation in 

O(1) rounds. 8Beyond clustering, Minimum Spanning Tree (MST) has attracted considerable interest in Euclidean space. For this problem, there is a notable gap between the value version and the solution version. For the value version, using the streaming algorithm that achieves an O(1 /ε 2)-approximation with space nε [CCJ +23], one can already obtain an O(1)-approximation in O(1) rounds. In contrast, it remains open to find an O(1)-approximate solution for Euclidean MST in 

O(1) rounds. Indeed, the currently best-known algorithms either find an O(1)-approximate MST in eO(log log n) rounds [JMNZ24], or achieve a polylog( n)-approximation in O(1) rounds [AAH +23]. Not surprisingly, when the dimension is low, say d = O(1), one can find an O(1)-approximation in 

O(1) rounds [ANOY14]. Other fundamental problems studied in this context include the Earth Mover Distance [ANOY14] and Max-Cut [MW26].

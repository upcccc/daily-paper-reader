## 摘要
在这项工作中，我们识别了机密联邦计算平台中的一系列侧信道，假设的内部攻击者可以利用这些侧信道来规避差分隐私（DP）保证。我们展示了 DP 如何缓解其中两个侧信道，其中一个已在我们的开源库中实现。

## Abstract
In this work, we identify a set of side-channels in our Confidential Federated Compute platform that a hypothetical insider could exploit to circumvent differential privacy (DP) guarantees. We show how DP can mitigate two of the side-channels, one of which has been implemented in our open-source library.

#### 1. 论文的核心问题与整体含义
本文探讨了在**机密联邦计算（Confidential Federated Compute, CFC）**平台中，即便使用了机密虚拟机（CVM，如 AMD SEV-SNP）和差分隐私（DP）技术，仍然存在的安全隐患。核心问题在于：**硬件隔离并非完全密封**，恶意的主机操作员（内部攻击者）可以通过观察**消息长度**和**内存分配**等侧信道信息，推断出受保护的敏感数据，从而绕过 DP 的隐私保证。研究动机是识别这些漏洞并提出基于 DP 的缓解方案，以增强分布式隐私计算系统的鲁棒性。

#### 2. 论文提出的方法论
论文针对两种主要的侧信道攻击提出了相应的防御机制：

*   **针对消息长度攻击的防御（Positive Laplace Mechanism）：**
    *   **核心思想：** 对 CVM 之间传输的加密消息进行填充（Padding），使消息长度不再直接反映数据内容。
    *   **技术细节：** 计算序列化字符串长度的**敏感度（Sensitivity）**，该敏感度取决于 SQL 查询结构、贡献限制和数据类型（如字符串长度限制）。
    *   **算法：** 采用“正向拉普拉斯机制”，在原始长度基础上加上一个经过偏移和截断的拉普拉斯噪声，确保填充后的长度始终不小于原始长度，并满足 $(\epsilon, \delta)$-DP。

*   **针对内存分配攻击的防御（StrictUDSAboveThreshold）：**
    *   **核心思想：** 针对动态数据结构（如 `std::unordered_map`）在达到容量阈值时会触发内存调整（Resize）产生的页错误（Page Faults）信号，通过随机化调整时机来掩盖数据特征。
    *   **技术细节：** 改进了经典的 **AboveThreshold** 算法，利用**单向敏感度（Unidirectional Sensitivity, UDS）**属性，将噪声规模从 $4/\epsilon$ 降低到 $2/\epsilon$。
    *   **算法流程：** 引入随机阈值，并在每次插入数据后，对比带有噪声的负载估算值与随机阈值，只有在超过阈值时才触发 Resize，并确保在物理容量耗尽前强制停止或调整。

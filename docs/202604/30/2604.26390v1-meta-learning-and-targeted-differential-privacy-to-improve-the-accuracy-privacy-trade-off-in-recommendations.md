## 摘要
在隐私保护推荐系统中，平衡差分隐私（DP）与推荐准确性是一项关键挑战，因为 DP 噪声会降低准确性。我们从数据和模型两个层面来解决这一权衡问题。在数据层面，我们仅对最可能泄露性别或年龄等敏感属性的典型用户数据应用 DP，以减少不必要的扰动；我们将其称为针对性 DP。在模型层面，我们利用元学习来提高对剩余 DP 噪声的鲁棒性。与标准方法相比，这实现了更好的准确性与隐私权衡：元学习提高了准确性，而与均匀应用 DP 和全量 DP 基准相比，针对性 DP 导致了更低的经验隐私风险。总之，我们的研究结果表明，在数据层面选择性地应用 DP 并结合模型层面的元学习，可以有效平衡推荐准确性与用户隐私。

## Abstract
Balancing differential privacy (DP) with recommendation accuracy is a key challenge in privacy-preserving recommender systems, since DP-noise degrades accuracy. We address this trade-off at both the data and model levels. At the data level, we apply DP only to the most stereotypical user data likely to reveal sensitive attributes, such as gender or age, to reduce unnecessary perturbation; we refer to this as targeted DP. At the model level, we use meta-learning to improve robustness to remaining DP-noise. This achieves a better trade-off between accuracy and privacy than standard approaches: Meta-learning improves accuracy and targeted DP leads to lower empirical privacy risk compared to uniformly applied DP and full DP baselines. Overall, our findings show that selectively applying DP at the data level together with meta-learning at the model level can effectively balance recommendation accuracy and user privacy.

*   **核心问题**：差分隐私（DP）虽然能保护用户隐私，但引入的随机噪声会显著降低推荐系统的准确性。
*   **研究动机**：传统的 DP 方法通常对所有用户数据均匀施加噪声，这会导致不必要的精度损失。研究者观察到，并非所有数据都同等程度地泄露隐私，只有那些具有“典型性（Stereotypical）”的数据（即能反映性别、年龄等敏感属性的偏好模式）才是泄露风险的主要来源。因此，如何在保护敏感信息的同时，最大限度地保留非敏感数据的效用，是本文的核心出发点。

### 2. 论文提出的方法论
该研究提出了一个双层优化框架，分别从数据层和模型层入手：
*   **数据层：针对性差分隐私（Targeted DP）**
    *   **核心思想**：仅对最能揭示敏感属性的“典型性”数据应用 DP 保护，而对非典型数据保持原样。
    *   **关键技术**：引入数据预算 $\beta$（表示不加保护的数据比例）。利用“项目-群体倾向性（IGI）”指标衡量每个评分项目对敏感属性（如性别）的指示强度。
    *   **流程**：计算用户 profile 中每个项目的典型性得分，筛选出前 $1-\beta$ 比例的最典型项目，对其应用“硬币翻转（Coin-flip）”机制（即以一定概率替换为随机评分），其余 $\beta$ 比例的数据直接用于训练。
*   **模型层：元学习（Meta-Learning）**
    *   **核心思想**：利用元学习模型（如 MetaMF）对噪声的鲁棒性来抵消 DP 引入的负面影响。
    *   **技术细节**：MetaMF 通过一个元网络为每个用户生成个性化参数。相比标准矩阵分解（MF），元学习能更好地从含有噪声的数据中提取跨用户的行为模式，从而在受到 DP 扰动的情况下维持较高的推荐准确率。

*   **针对性 DP 的有效性**：针对性 DP 能够在不额外损失准确性的前提下，比随机 DP 更有效地降低经验隐私风险。
*   **“甜蜜点（Sweet Spot）”**：实验发现当 $\beta=0.3$ 时（即保护 70% 的典型数据），用户数据的典型性降至最低，此时隐私风险甚至低于全量 DP（$\beta=0$），因为全量随机替换可能引入新的典型性特征。
*   **权衡优化**：通过调节 $\beta$，系统可以在形式化 DP 保证、经验隐私风险和推荐准确性之间找到最佳平衡。

## 摘要
大语言模型（LLMs）已成为合成数据生成的强大工具。一个尤为重要的应用场景是生成私有文本的合成副本，这需要仔细权衡隐私与效用。我们提出了“逼真且保护隐私的合成数据生成”（RPSG），该方法利用包括形式化差分隐私（DP）在内的隐私保护机制，以及包含个人信息的私有种子文本，来生成逼真的合成数据。与最先进的私有合成数据生成方法进行的全面实验表明，RPSG 在提供强大隐私保护的同时，实现了对私有数据的高保真度。

## Abstract
Large language models (LLMs) have emerged as a powerful tool for synthetic data generation. A particularly important use case is producing synthetic replicas of private text, which requires carefully balancing privacy and utility. We propose Realistic and Privacy-Preserving Synthetic Data Generation (RPSG), which leverages privacy-preserving mechanisms, including formal differential privacy (DP); and private seeds, in particular text containing personal information, to generate realistic synthetic data. Comprehensive experiments against state-of-the-art private synthetic data generation methods demonstrate that RPSG achieves high fidelity to private data while providing strong privacy protection.

### 论文总结：私有种子，公共大语言模型（RPSG）

#### 1. 核心问题与整体含义
论文探讨了在利用大语言模型（LLM）生成合成数据时，如何平衡**数据实用性（Utility）**与**隐私保护（Privacy）**。研究动机在于，许多具有科研价值的私有文本（如临床记录、社交媒体自我披露）因隐私风险无法共享。现有的微调方法（如 DP-SGD）计算成本高且难以应用于仅提供 API 的闭源模型，而纯提示词方法（Prompt-based）生成的数据质量往往较差。RPSG 旨在通过“私有种子”引导公共 LLM，在提供形式化差分隐私（DP）保证的同时，生成高保真度的合成副本。

#### 2. 方法论
RPSG 框架分为三个主要阶段：
*   **阶段 1：抽象与 DP 选择（Abstraction & DP Selection）**
    *   首先对私有种子文本进行 PII（个人身份信息）过滤。
    *   使用抽象模型（如 BART）为每个种子生成多个候选摘要，并结合情感对齐（Sentiment Alignment）确保表达意图一致。
    *   **核心技术：** 应用**高斯机制（Gaussian Mechanism）**。根据候选文本与原始种子的语义相似度计算得分，并加入校准后的噪声，从中选择一个满足 $(\epsilon, \delta)$-差分隐私要求的候选文本作为“安全种子”。
*   **阶段 2：合成数据生成（Synthetic Generation）**
    *   利用公共 LLM（如 GPT-4, Phi-4）根据上述安全种子生成多个合成变体。
*   **阶段 3：数据精炼（Data Refinement）**
    *   **NLL 过滤：** 训练一个代理模型计算合成样本的负对数似然（NLL）。移除 NLL 过低的样本（这些样本通常意味着模型发生了记忆，存在泄露风险）。
    *   再次进行 PII 脱敏，输出最终的合成数据集。

#### 3. 实验设计
*   **数据集：** 
    1.  **PubMed：** 医学摘要基准数据集。

*   **对比方法（Baselines）：**
    *   **DP-SGD：** 基于梯度的差分隐私微调方法。
    *   **AUG-PE：** 基于提示词的合成数据增强方法。
    *   **RUPTA：** 一对一的文本重写匿名化方法。
*   **评估指标：** 下游任务准确率（Next-word prediction）、情感对齐度、词汇多样性（Self-BLEU）、语义相似度（FID, Mauve）、成员推理攻击（MIA）抵抗力及 PII 泄露率。

实验设计非常充分且具有客观性：
*   **多模型覆盖：** 测试了包括 GPT-3.5, GPT-4o-mini, DeepSeek-R1, Phi-4, DistilGPT2 等多种规模和类型的模型。

*   **消融实验：** 专门验证了 NLL 过滤对抵抗成员推理攻击（MIA）的必要性，以及采样温度和样本量对性能的影响。

#### 6. 主要结论与发现
*   **实用性领先：** RPSG 生成的数据在下游任务准确率上显著优于 DP-SGD 和 AUG-PE。在 Reddit 数据集上，其准确率甚至比 DP-SGD 高出一倍以上。
*   **隐私保护强：** RPSG 在成员推理攻击下的 AUC 接近 50%（即攻击者几乎只能随机猜测），表现出极强的抗推理能力。
*   **情感与语义保真：** 尽管经过了 DP 处理，RPSG 仍能保持 90% 以上的情感对齐度，且在 FID 和 Mauve 等分布相似度指标上表现优异。

#### 7. 优点
*   **形式化隐私保证：** 结合了数学上的差分隐私证明，而非仅依赖启发式的脱敏。
*   **模型无关性：** 能够利用闭源 API 模型（如 GPT-4）生成高质量数据，解决了无法获取模型梯度时的隐私生成难题。
*   **端到端流程：** 从 PII 过滤到 NLL 筛选，构建了一个完整的隐私闭环。

#### 8. 不足与局限
*   **阈值敏感性：** NLL 过滤和相似度筛选的阈值需要针对不同模型进行精细调优，缺乏完全自动化的通用标准。
*   **聚合风险：** 论文指出目前的评估主要针对单样本泄露，尚未深入探讨攻击者通过聚合大量合成样本进行“群体画像（Re-profiling）”的风险。
*   **应用限制：** 对于极度稀缺或结构极其复杂的文本，抽象阶段可能会丢失过多关键细节。

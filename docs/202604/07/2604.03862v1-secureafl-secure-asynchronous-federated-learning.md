## 摘要
联邦学习（FL）使多个客户端能够在不共享私有训练数据的情况下，通过服务器协作训练全局机器学习模型。在传统联邦学习中，系统遵循同步方法，服务器在聚合模型更新以更新全局模型之前，需等待来自众多客户端的模型更新。然而，同步联邦学习受到掉队者（straggler）问题的阻碍。为了解决这一问题，异步联邦学习架构允许服务器在收到任何客户端的本地模型更新后立即更新全局模型。尽管具有优势，但异步联邦学习的去中心化特性使其容易受到投毒攻击。虽然已经提出了一些针对异步联邦学习的防御措施，但这些机制仍然容易受到高级攻击，或者依赖于不切实际的服务器假设。在本文中，我们介绍了 SecureAFL，这是一个旨在保护异步联邦学习免受投毒攻击的创新框架。SecureAFL 通过检测并丢弃异常更新，同时估计缺失客户端的贡献，从而提高异步联邦学习的鲁棒性。此外，它利用拜占庭容错聚合技术（如逐坐标中位数）来整合接收到的和估计的更新。在各种真实世界数据集上的广泛实验证明了 SecureAFL 的有效性。

## Abstract
Federated learning (FL) enables multiple clients to collaboratively train a global machine learning model via a server without sharing their private training data. In traditional FL, the system follows a synchronous approach, where the server waits for model updates from numerous clients before aggregating them to update the global model. However, synchronous FL is hindered by the straggler problem. To address this, the asynchronous FL architecture allows the server to update the global model immediately upon receiving any client's local model update. Despite its advantages, the decentralized nature of asynchronous FL makes it vulnerable to poisoning attacks. Several defenses tailored for asynchronous FL have been proposed, but these mechanisms remain susceptible to advanced attacks or rely on unrealistic server assumptions. In this paper, we introduce SecureAFL, an innovative framework designed to secure asynchronous FL against poisoning attacks. SecureAFL improves the robustness of asynchronous FL by detecting and discarding anomalous updates while estimating the contributions of missing clients. Additionally, it utilizes Byzantine-robust aggregation techniques, such as coordinate-wise median, to integrate the received and estimated updates. Extensive experiments on various real-world datasets demonstrate the effectiveness of SecureAFL.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：传统的同步联邦学习（FL）受限于“掉队者”问题（即服务器需等待所有客户端完成训练），导致效率低下。异步联邦学习（AFL）允许服务器在收到任何单一更新后立即更新全局模型，显著提升了系统扩展性和资源利用率。
*   **核心问题**：AFL 的去中心化和即时更新特性使其极易受到**投毒攻击**（包括无特定目标攻击和后门攻击）。现有的 AFL 防御机制要么鲁棒性不足，要么依赖于不切实际的假设（如服务器必须拥有一个干净的信托数据集）。
*   **研究目标**：开发一种无需信托数据集、能有效识别恶意更新并处理客户端更新滞后（Staleness）的安全异步联邦学习框架。

### 2. 论文提出的方法论
SecureAFL 的核心思想结合了**异常检测、缺失贡献估计和鲁棒聚合**三个阶段：
*   **本地模型更新过滤（Filtering）**：利用**Lipschitz 连续性**原理。服务器计算当前更新与该客户端历史更新之间的 Lipschitz 因子（平滑度）。如果该因子超过了历史分布的特定百分位阈值（如 80%），则认为该更新存在异常偏差，予以剔除。
*   **本地模型更新估计（Estimation）**：为了在异步环境下进行有效的鲁棒聚合，服务器需要参考其他未上传客户端的状态。SecureAFL 利用 **L-BFGS 算法**和历史梯度轨迹，通过近似积分海森矩阵（Hessian Matrix）来估算那些在本轮未提交更新的客户端的当前可能更新。
*   **鲁棒聚合（Aggregation）**：服务器将接收到的（且通过过滤的）更新与所有估算的更新组合在一起，应用**逐坐标中位数（Coordinate-wise Median）**等拜占庭容错聚合规则。这种方法确保了即使估算的更新中包含恶意客户端的轨迹，其影响力也会被中位数规则抑制。

*   **高鲁棒性**：在所有测试的投毒攻击下，SecureAFL 的测试错误率（TER）和攻击成功率（ASR）均显著低于现有防御方法。

### 7. 优点
*   **无需先验假设**：不要求服务器拥有信托数据集，这在隐私敏感的实际应用中更具可行性。
*   **动态估计机制**：通过 L-BFGS 估计缺失更新，解决了异步 FL 中单次更新无法进行统计学异常检测的难题。
*   **理论支撑**：论文提供了非凸目标函数下的收敛性证明，保证了算法在数学上的完备性。

### 8. 不足与局限
*   **存储开销**：服务器需要存储每个客户端的历史更新差异和全局模型差异（$O(n\epsilon d)$），对于超大规模模型或海量客户端，内存压力会增大。
*   **潜在隐私风险**：虽然不直接访问原始数据，但服务器通过历史轨迹估计客户端更新，在“诚实但好奇”的服务器模型下，可能比传统聚合方法泄露更多关于客户端本地目标函数演化的信息。
*   **参数敏感性**：虽然实验显示对过滤阈值 $\alpha$ 不太敏感，但在极端动态的网络环境下，如何自动调整该阈值仍是一个挑战。

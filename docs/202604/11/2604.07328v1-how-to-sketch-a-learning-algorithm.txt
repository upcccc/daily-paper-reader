# How to sketch a learning algorithm 

# Sam Gunn April 9, 2026 

Abstract 

How does the choice of training data influence an AI model? This broad question is of central importance to interpretability, privacy, and basic science. At its technical core is the data deletion problem: after a reasonable amount of precomputation, quickly predict how the model would behave in a given situation if a given subset of training data had been excluded from the learning algorithm. We present a data deletion scheme capable of predicting model outputs with vanish-ing error Œµ in the deep learning setting. Our precomputation and prediction algorithms are only poly(1 /Œµ ) factors slower than regular training and inference, respectively. The storage requirements are those of poly(1 /Œµ ) models. Our proof is based on an assumption that we call stability . In contrast to the assumptions made by prior work, stability appears to be fully compatible with learning powerful AI models. In support of this, we show that stability is satisfied in a minimal set of experiments with microgpt. Our code is available at microgpt-sketch. At a technical level, our work is based on a new method for locally sketching an arithmetic circuit by computing higher-order derivatives in random complex directions. Forward-mode automatic differentiation allows cheap computation of these derivatives.

21 Introduction 

One of the most fundamental problems in AI is to understand model behavior as a function of training data. Variations on this problem abound, going under names such as datamodel-ing, influence functions, data valuation, and machine unlearning. While the exact problem formulation depends on the application of interest, they are all addressed by data deletion .1

A data deletion scheme consists of two algorithms: 

‚Ä¢ precomputation , which takes as input the description of a learning algorithm A and a training dataset T , and outputs the trained model A(T ) together with some arbitrary auxiliary information; and 

‚Ä¢ prediction , which takes as input the description of a function œï of the model parameters (called a ‚Äúmeasurement‚Äù) together with a small subset D of the training dataset, and uses the precomputed auxiliary information to quickly predict œï(A(T \ D)): i.e., what the measurement outcome on the learned model would have been if the subset D had been excluded from the training dataset .Consider the following scenario. Suppose an artist posts their work online, and this work is then included in the training data for a new AI model. This model later generates some con-tent x that appears related to the artist‚Äôs work. How can we know whether the relationship is causal? If the model was trained using the precomputation algorithm of a data deletion scheme, then we can quickly compute the probability that the model would have produced the given content if all of the artist‚Äôs work were excluded from the set of training data. All we need to do is search the training data for the artist‚Äôs work and query the prediction algorithm using the measurement that asks ‚Äúwhat is the probability that this model will produce x?‚Äù Although they would be extremely useful, data deletion schemes are notoriously hard to come by. There currently does not exist an efficient scheme capable of achieving arbitrary prediction error Œµ for actual AI algorithms‚Äîin theory or in practice. We believe this is largely due to a lack of good abstractions, making it difficult to understand the set of available algorithmic tools. As a result, existing data deletion schemes rely on surrogate models or heuristics and often only weakly correlate with the ground truth in practice [IE25]. In this work, we demonstrate that this situation may not be inevitable. We introduce a simple assumption about the learning algorithm which we call stability and use it to provably solve the data deletion problem with vanishing prediction error Œµ. We find that our assumption is practically reasonable using a small set of experiments with microgpt [Kar26]. Stability is an assumption about both the learning algorithm A and a given measurement 

œï. Roughly, it quantifies the sensitivity of œï(A(T )) to small changes in T . The more robust

3the measurement is to changes in training data, the more stable it is. Our data deletion scheme has the following properties: 

‚Ä¢ precomputation is only a poly(1 /Œµ ) factor more expensive than training the model; 

‚Ä¢ predicting the model behavior after deleting d examples is only a d ¬∑ poly(1 /Œµ ) factor more expensive than evaluating the trained model once; 

‚Ä¢ the precomputed data can be stored in the space of poly(1 /Œµ ) trained models; and 

‚Ä¢ under the weakest form of our stability assumption, the prediction error is O(Œµ) for any constant number of deletions d, while a stronger form of our stability assumption bounds the prediction error for arbitrary d.Our techniques are based on a new method of locally sketching an arithmetic circuit, which may be of independent interest.

Upgrading to data deletion. So far we have assumed that precomputation knows œï,which is not allowed in data deletion. This nonetheless constitutes a local sketching method for stable arithmetic circuits, and is presented in Section 4. It serves as the main technical component behind our data deletion scheme in Section 5. In our data deletion scheme, precomputation works identically except that f is replaced by 

A. Even though A has many outputs, this does not increase the computational complexity: Precomputation can still be done in time roughly k s ¬∑ size (A) using forward-mode automatic differentiation. Prediction, on the other hand, becomes more complicated. While it would be natural to simply compute the analogous approximation to A(1D) as was done for f (1D), and to then apply œï to this approximation, this is not guaranteed to be accurate by stability alone. For instance, imagine that A performed some arbitrarily unstable computation, which was then undone by œï so that f = œï ‚ó¶ A is a constant function. Then f is highly stable, but any approximation error on A(1D) translates to large error on the output of œï.Instead, our prediction algorithm will use the Fa` a di Bruno formula to push the derivatives of A through œï, thereby obtaining the exact same sketch of f that we would have obtained if we had applied precomputation directly to f . This involves computing multi-directional forward-mode derivatives, which can be done with fast subset convolution [BHKK07] (see Section 3.1). Interestingly, the final complexity of our data deletion scheme stated in our main theorem, Theorem 2, would be worse if we did not use fast subset convolution to compute forward-mode derivatives. We are optimistic that it may be possible to remove the need for the Fa` a di Bruno formula. While it does not significantly affect the asymptotic complexity stated in Theorem 2, it may be the most practically inefficient aspect of our data deletion scheme. 

Minor differences from the main body. We have glossed over several subtleties for the purposes of this introduction, creating differences from the main body of the paper. To minimize confusion, we briefly explain the main differences now. 8First, the body considers the learning and measurement algorithms as arbitrary arithmetic circuits; our restriction to stochastic gradient descent in the introduction is not important. After Section 1, we do not even return to the ‚Äúlearning algorithm‚Äù and ‚Äúmeasurement func-tion‚Äù terminology until we get to our data deletion scheme in Section 5. Our sketching algorithm presented in Section 4 is accordingly more general than suggested in the introduc-tion. It applies equally well to high-dimensional outputs, and can be used to locally sketch an arithmetic circuit around an arbitrary input x‚àó as long as the circuit is stable at x‚àó.Second, we find it convenient to make most statements over the complex numbers C rather than R. Since our model for AI computations is simply arithmetic circuits, this does not cost us any generality. In any case, as explained in Section 2.2, this choice is not practically important and does not hinder our implementation in Section 5.2. Third, the definition of stability presented in Equation (1) is less general than the one we actually use in the body. The real definition, presented in Section 3.3, is more conducive to proving quantitative bounds on the error of our schemes. Fourth, we are unable to show that an average of estimates from Equation (2) concentrates well. We would therefore obtain a poor bound on the failure probability if we used the scheme from this introduction. Instead, we use a median-of-means estimator in our prediction algorithms of both Sections 4 and 5. We suspect that the median-of-means estimator is not necessary, and we hope that future work can prove this without sacrificing our parameters. 

# 1.2 Related work 

Variations on the data deletion problem abound, and we only survey some of the most relevant work here. Our work is most similar to the line of work on datamodeling or predictive data attribution , although there are several key differences which we outline below. For an introduction to the area, we recommend the tutorial of [IGEP24]. A surrogate model is an idealized model of a system which is used in place of the actual system. Every prior work with provable guarantees for data deletion in the deep learning setting relies on a surrogate model. This precludes a vanishing prediction error, as any method that relies on a surrogate model necessarily incurs the fixed approximation error of the surrogate model itself. In contrast, our stability assumption is not a surrogate model: It appears to be fully compatible with learning powerful models on large datasets. 

The additivity assumption. Perhaps the most successful surrogate model for answering questions of data influence has been the additivity assumption. This assumption says that the function f mapping the list of data weights to a model prediction (on a fixed test query) is linear. In other words, it is assumed that the influence of a set of training data on any model prediction is the sum of the influences of the individual training examples. For instance, the work on datamodeling [IPE +22, PGI +23, IE25] and influence functions 

[HRRS86, KL17, KATL19] uses the additivity assumption. 9In reality, it is of course understood that additivity does not hold for real models. The hope is rather that the additivity assumption does not incur too large an error, at least for small perturbations in the data weights. There are theoretical reasons to believe that the additivity assumption should incur bounded error for random perturbations [IPE +22, SGBA23], but in practice it is usually necessary to significantly alter learning algorithms in order to keep this error small [IE25]. Moreover, random perturbations may not capture many of the most important use cases such as removing many closely-related examples. Designing data deletion schemes with the additivity assumption has several major drawbacks: 

‚Ä¢ the prediction error becomes unreasonable past a certain number of deletions; 

‚Ä¢ there is no way to reduce the prediction error by expending more computation; 

‚Ä¢ even if additivity is a good approximation under random deletions, it can be a poor approximation for worst-case deletions. We therefore view our departure from the additivity assumption as an essential aspect of this work. However, we emphasize that even assuming additivity , our scheme greatly improves on prior work. The strongest result in the additivity model is the work of [IE25], which exactly computes counterfactuals assuming additivity. Accepting additivity, the main drawback of that work is that it is necessary to know the measurement function ahead of time during precomputation. Our work yields the first scheme in the deep learning setting to both (a) not require foreknowledge of the measurement function during precomputation and (b) guarantee a provable bound on the counterfactual prediction error. 

Post hoc certified unlearning. It is common for machine unlearning schemes to make no reference to the algorithm used to train the model. For instance, one method is based on performing gradient ascent on the loss for the set of data to be deleted [TKP +24]. These methods can be applied to ‚Äúdelete‚Äù even data that was never seen during training of the model. We discuss this approach in more detail in Appendix A. 

Fourier analysis. As mentioned in the paragraph on the additivity assumption, [SGBA23] argue that the additivity assumption should be a good approximation for the influence of random subsets of training data. They adopt the ‚Äúdiscrete influence‚Äù perspective, where one views the datamodeling function as being restricted to the hypercube, f : {0, 1}n ‚Üí R,allowing them to use ideas from the analysis of Boolean functions. It would be interesting to see whether similar arguments can be used to shed light on our stability assumption. It would also be interesting to see whether our techniques can be extended to the discrete influence setting. This could potentially enable data deletion using a stability assumption related to the Fourier spectrum of the discrete influence function. 10 Other work. Since we are not focused on any particular application of data deletion, we do not discuss considerations such as privacy concerns when applying data deletion as a solution to machine unlearning. We refer the interested reader to [GGV20, CSSV23]. We caution that data deletion by itself is not a complete solution to machine unlearning, and na¬® ƒ±ve applications of data deletion can in some cases actually harm privacy [CZW +21, CJZ +22, CS23]. There is also work on provable data deletion for particular algorithms such as k-means clus-tering [GGVZ19], linear classifiers [GGHvdM20], and linear or logistic regression [ISCZ21]. 11 2 Preliminaries

# 2.2 Arithmetic circuits and machine learning 

We make the pedagogical decision to model machine learning computations‚Äîboth training and inference‚Äîas arithmetic circuits over the complex numbers. All of our results apply essentially identically to real computations including general gates such as ReLU, GELU, or Softmax; the technical details would just be more cumbersome. While these gates may not be defined on complex inputs, our method only requires that they have local polynomial approximations around real inputs. Indeed, we easily implemented the general case for microgpt (see Section 5.2). Formally, an arithmetic circuit is a function f : Cn ‚Üí Cp defined by a directed acyclic graph with labeled nodes. Each node in the graph with in-degree zero is labeled by either an input index i ‚àà [n] or a complex number. Each node with positive in-degree is labeled by either + or √ó. The output of a gate is passed as input to all of its children. We do not restrict the in-degree or out-degree. Each node with out-degree zero is further labeled by an output index i ‚àà [p]. Whereas input indices may be re-used, each output index is a label for exactly one node. For x ‚àà Cn, the output f (x) ‚àà Cp is computed by replacing each input-index node label i ‚àà [n] with the corresponding input value xi, following the computation specified by the graph, and outputting the vector y ‚àà Cp specified by the output values at the output indices. We define the size of f , denoted size (f ), as the maximum of n, p, and the number of edges in the graph. Since we count real operations as constant time, evaluating f requires 

O(size (f )) operations. In the context of machine learning, note that if model evaluations can be described by arithmetic circuits acting on model parameters, then the Baur-Strassen theorem [CKW11, Theorem 9.10] implies that gradient steps can also be represented by arithmetic circuits. Therefore it is without further loss of generality to assume that training, in addition to inference, is an arithmetic circuit. 13 3 The toolkit

# 3.1 Forward-mode automatic differentiation 

Reverse-mode automatic differentiation (i.e., backpropagation) can compute the gradient of a circuit f : Cn ‚Üí C in time O(size (f )). This section begins with a discussion of the data deletion problem. We then explain our scheme in Section 5.1, and we conclude with our experiments on microgpt in Section 5.2. A data deletion scheme allows us to quickly predict model behavior after deleting training data. It consists of two phases: 1. precomputation , which is allowed to expend resources comparable to those required to train the entire model; and 2. prediction , which should use the precomputed data to perform inference as if the model was trained without a given subset of training data. Using random access to the precomputed data, prediction should run in time much faster than re-training. Ideally, it should run in time independent of the number of training exam-ples n.We assume that training and inference are deterministic, which is without loss of generality as one can always fix the randomness.
